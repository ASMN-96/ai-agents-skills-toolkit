# Commit Message Template

```text
<type>: <concise outcome>

Why:
- <business, product, safety, or maintenance reason>

What:
- <focused implementation summary>

Validation:
- <commands or checks actually run>
- <skipped or unavailable checks, if any>
```

Allowed types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `security`, `release`.

Do not include secrets, credentials, client names, pricing, private strategy, proprietary roadmap, or local user paths.
