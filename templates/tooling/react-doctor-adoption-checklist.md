# React Doctor Adoption Checklist

Owner must review before applying. Codex must not claim output unless React Doctor actually ran and produced current output. Package-manager changes require separate approval.

## Adoption Scope

- Confirm the project is React or React Native.
- Confirm ESLint, React Hooks, typecheck, tests, and browser gates remain baseline.
- Decide whether React Doctor is local-only, script-only, CI, GitHub Action, or PR writer.
- Require separate approval for GitHub Action setup, PR write permissions, or agent skill install.
- Decide whether findings are advisory, warning, or blocking.
- Define rollback and removal path.
