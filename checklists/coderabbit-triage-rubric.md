# CodeRabbit Triage Rubric

Use this rubric when automated PR review comments need classification.

## Classification

- Required blocker: correctness, security, data exposure, failing tests, release risk, or required project policy.
- Fix in scope: small scoped improvement that clearly reduces risk without expanding the task.
- Clarify: comment is ambiguous, conflicts with local context, or needs owner decision.
- Defer: valid hardening but outside the approved scope.
- No action: incorrect, duplicate, stylistic-only, or already handled.

## Required Evidence

- PR/check status was inspected when available.
- Required comments are linked or summarized.
- Fixes stay within the approved scope.
- Deferred comments include the reason.
- Merge is not recommended while required checks or required comments remain unresolved.
