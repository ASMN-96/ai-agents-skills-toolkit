# Premium UI/UX Review

Use this as the public-safe UI/UX quality skill for frontend, dashboard, wizard, app, and browser-visible experiences.

This skill reviews usability and visual quality. It does not authorize package changes, framework changes, CI edits, MCP setup, global config changes, backend/security changes, or claims of browser verification without observed evidence.

## Review Focus

- Clarity of primary workflow and user intent.
- Responsive layout, mobile behavior, accessibility, and keyboard usability.
- Visual hierarchy, spacing, density, typography, interaction states, and empty/error/loading states.
- Design-system consistency and avoidance of decorative or inconsistent patterns.
- Evidence quality: screenshots, browser checks, or manual QA must be labeled honestly.

## Completion Evidence

Report scope, UI/UX findings, files or routes reviewed, browser/manual evidence if actually collected, skipped checks, and remaining QA.
