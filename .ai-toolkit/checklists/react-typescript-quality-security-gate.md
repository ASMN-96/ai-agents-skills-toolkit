# React TypeScript Quality Security Gate

- TypeScript strictness is preserved.
- Weak typing and suppressions are justified.
- React hooks rules and dependency arrays are reviewed.
- Loading, error, empty, disabled, and async states are covered.
- Behavior changes have focused tests or explicit manual QA.
- No package, lockfile, CI, MCP, or global config changes occur without separate approval.
- AI-generated code is checked for hidden rewrites, duplicate abstractions, hardcoded IDs, weak errors, and untested critical paths.
