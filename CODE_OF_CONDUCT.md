# Code of Conduct

This project expects professional, respectful collaboration.

## Expected Behavior

- Be direct, factual, and constructive.
- Respect security, privacy, and project boundaries.
- Assume changes may affect real users, maintainers, reviewers, and downstream projects.
- Surface uncertainty, risks, and limitations clearly.

## Unacceptable Behavior

- Harassment, discrimination, threats, or personal attacks.
- Publishing private data, secrets, tokens, credentials, local paths, or confidential project details.
- Pressuring maintainers to bypass security, validation, review, or release gates.
- Misrepresenting skipped, mock, dry-run, metadata-only, or unavailable checks as real execution.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or limit participation when behavior puts users, maintainers, security, or project integrity at risk.
